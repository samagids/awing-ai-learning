# torchaudio is a HARD transitive requirement: coqui-tts's TTS/__init__.py
# unconditionally imports the XTTS config chain (xtts_config.py → xtts.py)
# which does `import torchaudio` at module load. Even VITS-only callers
# can't import `TTS.*` without it.
#
# CRITICAL: torchaudio's compiled libtorchaudio.pyd links against specific
# torch DLLs at load time. THREE things must match between torch and
# torchaudio or Windows reports
#   "Could not find module ... libtorchaudio.pyd (or one of its dependencies)"
# or "[WinError 127] The specified procedure could not be found" — both are
# missing-symbol errors in the torch DLLs that torchaudio imports:
#   1. MAJOR.MINOR.PATCH version (2.5.1 ≠ 2.4.1 ≠ 2.7.0 ≠ 2.11.0)
#   2. Build flavor: +cpu vs +cu124 vs +cu128 (CPU torch CANNOT load CUDA
#      torchaudio — they reference different DLL symbols entirely)
#   3. Python version of the wheel (cp311 vs cp312)
#
# FOURTH constraint, learned the hard way in Session 54:
#   coqui-tts 0.27 still calls `torchaudio.info(path).num_frames` from
#   TTS/tts/datasets/dataset.py at module load of every training run.
#   torchaudio 2.8+ refactored I/O around torchcodec and dropped `info`
#   as a top-level attribute → AttributeError before step 0.
#   So: torchaudio MUST be ≤ 2.7.x for any Coqui TTS training to start.
#
# FIFTH constraint, also Session 54:
#   RTX 50-series (Blackwell, sm_120) needs CUDA 12.8 kernel images.
#   Standard cu124 wheels only ship sm_50–sm_90. Result: torch loads,
#   torch.cuda.is_available() returns True, but the FIRST kernel launch
#   crashes with "no kernel image is available for execution on the
#   device". cu128 builds added Blackwell support starting at torch 2.7.0.
#
# Pinning intersection that satisfies BOTH constraints (Coqui + Blackwell):
#   torch 2.7.0+cu128 AND torchaudio 2.7.0+cu128
#
# Install procedure (always --no-deps to avoid OneDrive file-lock cascades
# on the multi-GB torch wheel):
#   pip install --force-reinstall "torch==2.7.0" "torchaudio==2.7.0" \
#       --index-url https://download.pytorch.org/whl/cu128 --no-deps
#
# Verify all four properties in one shot:
#   python -c "import torch, torchaudio; print('torch:', torch.__version__); \
#       print('torchaudio:', torchaudio.__version__); \
#       print('CUDA:', torch.cuda.is_available()); \
#       print('Capability:', torch.cuda.get_device_capability(0)); \
#       print('has info:', hasattr(torchaudio, 'info'))"
#
# Want: torch 2.7.0+cu128, torchaudio 2.7.0+cu128, CUDA True,
#       Capability (12, 0) with NO sm_120 warning, has info True.
#
# Older / non-Blackwell GPU paths (kept for reference):
#   torch 2.5.1+cu124   → torchaudio==2.5.1 from .../whl/cu124
#   torch 2.6.0+cu124   → torchaudio==2.6.0 from .../whl/cu124
#   torch 2.11.0+cpu    → torchaudio==2.11.0 from .../whl/cpu
#                          (CPU-only — works but ~15× slower than GPU)
#
# Bare names below intentionally — pip --no-deps install above sets the
# real version, and listing a hard pin here causes pip to RE-RESOLVE on
# every plain `pip install -r` and clobber the cu128 build with cu124 or
# cpu. Keep these unpinned so pip respects whatever's already installed.
torchaudio
